# Screenshot Checklist

This page lists screenshots needed for the documentation. 

## How to Take Screenshots

1. Use browser developer tools to set consistent viewport (1280Ã—800)
2. Use light mode unless documenting dark mode features
3. Save as PNG with descriptive filename
4. Place in `/docs/public/screenshots/`

## Naming Convention

```
{section}-{feature}-{description}.png
```

Examples:
- `admin-sysreg-main.png`
- `stepper-events-add.png`
- `dashboard-navigation.png`

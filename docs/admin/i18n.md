## How i18n Works

The i18n system uses:
1. **Translation keys** stored in sysreg tables
2. **`useI18n` composable** for frontend access
3. **Language detection** based on user/project settings

## Translation Structure

Translations are organized by:
- **Entity type** (buttons, labels, messages, etc.)
- **Language code** (de, en, etc.)

### Example Keys

```
button.save = "Speichern" (de) / "Save" (en)
button.cancel = "Abbrechen" (de) / "Cancel" (en)
label.name = "Name" (de/en)
message.success = "Erfolgreich gespeichert" (de) / "Successfully saved" (en)
```

### Adding New Translations

1. Click **"+ Add Translation"**
2. Enter the key name
3. Add values for each supported language
4. Save

## Language Detection

The system detects language in this order:

1. **User preference** (if logged in)
2. **Project setting** (project-specific language)
3. **Browser language** (Accept-Language header)
4. **Default** (German for most content)

## Using i18n in Code

```typescript
import { useI18n } from '@/composables/useI18n'

const { t, button, label } = useI18n()

// Usage
button('save')     // → "Speichern" or "Save"
label('name')      // → "Name"
t('custom.key')    // → Custom translation
```

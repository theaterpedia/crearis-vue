# Schritt 6: Check-in

Im letzten Schritt **aktivierst** du dein Projekt.

::: info Nur für Projekt-Owner
Dieser Schritt ist nur sichtbar, wenn du **Owner** des Projekts bist.
:::

## Projekt-Status

Dein Projekt durchläuft verschiedene **Status-Stufen**:

| Status | Beschreibung |
|--------|--------------|
| **new** | Gerade erstellt, im Stepper-Modus |
| **demo** | Test-/Demo-Modus |
| **draft** | Entwurf, nach Aktivierung |
| **published** | Öffentlich sichtbar |

## Vor der Aktivierung

Prüfe, ob alles vorbereitet ist:

### Checkliste

- [ ] Mindestens ein **Event** angelegt
- [ ] Projekt-**Beschreibung** vorhanden
- [ ] **Bilder** hochgeladen
- [ ] **Team** konfiguriert (optional)
- [ ] **Design** ausgewählt (optional)

## Projekt aktivieren

<div class="screenshot-placeholder">
Screenshot: "Projekt aktivieren" Button
</div>

1. Klicke auf **"Projekt aktivieren"**
2. Der Status wechselt zu **"draft"**
3. Du verlässt den Stepper-Modus
4. Das Dashboard wird angezeigt

## Was passiert bei der Aktivierung?

### Stepper → Dashboard

Nach der Aktivierung:
- Der **Stepper** (links) verschwindet
- Die **Navigation** erscheint mit Tabs:
  - Homepage
  - Events
  - Posts
  - Images
  - Users
  - Theme
  - Layout
  - Navigation
  - Pages

### Projekt-URL

Dein Projekt erhält eine öffentliche URL:
```
https://theaterpedia.org/sites/{projekt-id}
```

## Nach der Aktivierung

Im **Dashboard-Modus** kannst du:
- Inhalte weiter bearbeiten
- Neue Events/Posts hinzufügen
- Das Design anpassen
- Zur Veröffentlichung vorbereiten

### Nächster Schritt: Veröffentlichen

Um dein Projekt **öffentlich sichtbar** zu machen:
1. Gehe zu den Projekt-Einstellungen
2. Ändere den Status zu "published"

::: tip
Teste dein Projekt gründlich im **draft**-Status, bevor du es veröffentlichst!
:::

## Strategic Decision: Strategy C as Primary

**Chosen Approach:** Strategy C (Participant Journey) as universal default

**Rationale:**
- **80% newcomers** – especially in years 1-2, everyone feels like a newcomer
- **20% institutions** – deep integration = custom implementation + webdesign work (not default)
- DASEI becomes **edge-case extension**, not the template

> Many small website-projects, interconnected, each with their own simple workflow.

## 🟡 HIGH: Polymorphic Routing (Comments/Workitems)

### Concept
```
/api/comments → polymorphic router
  ├─ entity_type='image'   → image_workitems (Migration 059)
  ├─ entity_type='project' → tasks (category='comment')
  ├─ entity_type='event'   → event_workitems (v0.6)
  └─ entity_type='post'    → comments (Migration 057)
```

### Why This Matters
- "Workitems" = interface/contract, NOT a single table
- Different entity types have different lifecycles
- Enables Odoo sync per entity type
